# SHACK-SERVER

# Project Charter

**Codename:** Project Horizon

> **Turning amateur radio data into actionable information.**

---

# Vision

SHACK-SERVER is an open-source Operations Center for modern amateur radio.

It combines information from radios, DX clusters, propagation services, logging software and external data sources into one intelligent platform.

Instead of simply displaying information, SHACK-SERVER analyses, prioritises and presents actionable insights that help operators make better decisions.

Our goal is to build the most modern, extensible and operator-focused amateur radio platform available.

---

# Mission

Create a professional, reliable and open-source platform that transforms amateur radio data into operational intelligence.

SHACK-SERVER should become the central information hub for every radio operator—whether operating from a home station, a portable setup, during contests or on DXpeditions.

---

# Product Values

## Connected

Bring together information from multiple independent systems into one unified platform.

Examples include:

- DX Clusters
- Radio CAT interfaces
- Propagation services
- Solar data
- Weather information
- Logging software
- Rotor controllers
- External APIs

---

## Intelligent

Raw data has limited value.

SHACK-SERVER enriches, filters and correlates information to generate meaningful operational knowledge.

---

## Actionable

The software should never overwhelm the operator.

Instead, it should answer one simple question:

> **What should I do next?**

---

## Reliable

The software must be stable enough for everyday operation.

Reliability always has a higher priority than adding new features.

---

# Design Philosophy

SHACK-SERVER follows a modern dark interface inspired by professional Network Operations Centers.

The interface is designed for long operating sessions and minimises visual distraction.

Every screen should immediately communicate what is important.

Colour is used intentionally:

- 🟢 Green — Normal operation
- 🟡 Orange — Attention required
- 🔴 Red — Immediate action required
- 🔵 Blue — Information

Animations should be subtle and always have a purpose.

---

# User Experience

The interface should never force the operator to search for important information.

Instead, SHACK-SERVER continuously answers questions such as:

- What is happening now?
- What should I work next?
- Which band is opening?
- Which DX is worth calling?
- Is my station ready?
- Are there any problems?

The interface supports decisions rather than simply displaying data.

---

# Architecture

SHACK-SERVER is designed as a modular platform.

Every major function is implemented as an independent module.

Examples include:

- DX Intelligence
- Propagation
- Station Monitoring
- Rotor Control
- Contest Assistant
- SOTA / POTA
- Satellite Tracking
- Analytics
- Notifications

New modules should integrate without affecting existing functionality.
